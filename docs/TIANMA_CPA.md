# 天马 / CPA 渠道匹配

账号查询的回复格式不变：

```text
查询结果：
账号：yizhihen
注册IP：112.36.175.232
注册省份：山东 济南
注册来源：天马/CPA
渠道编码：J60-15
状态：已匹配
```

## 推荐表格格式

为天马 / CPA 新建一个映射工作表，至少包含两列：

| 渠道编码 | 注册来源 |
| --- | --- |
| J60-15 | 天马/CPA |

也可以把 `注册来源` 列名写成 `来源编号`。如果使用 `来源编号`，仍然兼容原来的规则：渠道编码在它左侧一列。

## 机器人配置

保留原来的 `sheet_url`，把天马 / CPA 映射表填到 `extra_sheet_urls`：

```json
{
  "sheet_url": "原来的来源编号表格链接",
  "extra_sheet_urls": "天马/CPA映射表链接",
  "service_account_file": "/config/service-account.json"
}
```

`extra_sheet_urls` 可以填写多个表格链接，每行一个。

## 注意

如果后台返回的注册来源就是 `天马/CPA`，表格里也要写完整的 `天马/CPA`。程序现在会保留这个斜杠，不会再误处理成 `天马`。

如果不同账号都返回同一个注册来源 `天马/CPA`，但实际要匹配不同渠道编码，仅靠注册来源无法区分。需要后台返回额外唯一字段，或者在映射表里改成不同的注册来源值。
